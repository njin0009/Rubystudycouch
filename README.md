# StudyCouch

StudyCouch is a focused AWS CLF-C02 exam practice app with a local-first quiz engine, progress tracking, mistake review, saved questions, notes, check-ins, and PDF reference support.

## Current MVP

- 719 CLF-C02 practice questions
- Random, domain, mistake, and saved-question practice modes
- Single-choice and multi-select question handling
- Local profile backup and import/export
- Mistake review, saved questions, notes, weekly progress, and study streaks
- PDF.js support for viewing the original PDF question bank

## Stack

- Frontend: React, TypeScript, Vite
- UI system: Tailwind CSS with shadcn-style components in `src/components/ui`
- Legacy quiz engine: migrated static HTML/CSS/JavaScript mounted inside React
- PDF.js for local PDF viewing
- Cloudflare Pages for frontend hosting
- Supabase for Auth, Postgres, Storage, and cross-device sync
- GitHub for version control and portfolio presentation

## File Guide

### Root Files

- `.gitignore`: Lists generated files, local-only files, and private files that should not be committed. Keep `node_modules`, `dist`, `.env`, and `PDF_DOCS` ignored.
- `README.md`: Main project guide. Update this whenever the project structure, setup commands, deployment steps, or major feature responsibilities change.

### Frontend: `apps/web`

- `apps/web/index.html`: Vite HTML entry. It should stay small and only define the root element, metadata, fonts, and the Vite script entry.
- `apps/web/public/favicon.svg`: Browser tab icon used by `index.html`.
- `apps/web/package.json`: Frontend dependencies and scripts. Update this when adding packages or changing commands.
- `apps/web/package-lock.json`: Locked dependency versions generated by `npm install`. Commit it so installs are reproducible.
- `apps/web/vite.config.ts`: Vite build configuration. Update this when changing build behavior, aliases, plugins, or deployment-specific settings.
- `apps/web/tsconfig.json`: TypeScript project reference entry. Usually rarely changes.
- `apps/web/tsconfig.app.json`: TypeScript rules for the app source. Update when changing TypeScript strictness, module settings, or included paths.
- `apps/web/.env.example`: Public template for required environment variables. Add new variable names here, but never put real secrets in it.
- `apps/web/README.md`: Frontend-specific maintenance guide. Update this when frontend folders, commands, migration strategy, or component ownership changes.

### Frontend Source: `apps/web/src`

- `apps/web/src/main.tsx`: React entry point. It mounts the app into `#root` and imports global styles.
- `apps/web/src/App.tsx`: Top-level app shell. Use this to introduce routing, layout, providers, or page-level composition.
- `apps/web/src/vite-env.d.ts`: Type definitions for Vite and browser globals. Update when legacy scripts or global browser APIs need TypeScript declarations.
- `apps/web/src/lib/supabase.ts`: Shared Supabase client configured from Vite environment variables. It also enforces Supabase `apikey` and bearer headers on outgoing Supabase requests.
- `apps/web/src/lib/supabaseHealth.ts`: Simple helper for checking whether the frontend can reach Supabase.
- `apps/web/src/lib/studySync.ts`: Shared study-data sync layer. It loads signed-in users' cloud study snapshot from Supabase, syncs attempts, saved questions, important marks, notes, and check-ins, and clears all study records when the user chooses Clean data.
- `apps/web/src/lib/utils.ts`: Shared `cn` helper for merging Tailwind/shadcn class names.
- `apps/web/src/components/ui/*`: shadcn-style reusable UI primitives and composed UI components. Keep shared UI here so imported components can use the default `@/components/ui/...` path.
- `apps/web/src/components/ui/animated-characters-login-page.tsx`: Animated login/register page connected to Supabase email/password auth and Google OAuth. Registration passwords must be 6-12 characters and include uppercase, lowercase, number, and symbol characters.
- `apps/web/src/features/auth/AuthPanel.tsx`: Supabase magic-link login, session display, profile upsert, and sign-out UI.
- `apps/web/src/features/auth/auth.css`: Styles for the fixed login panel.
- `apps/web/src/styles/legacy.css`: Existing visual system migrated from the original HTML. Edit this for current UI styling until components get their own styles.
- `apps/web/src/legacy/LegacyStudyCoach.tsx`: Temporary React wrapper that mounts the old quiz app and loads legacy scripts. Gradually shrink this file as features are rewritten in React.
- `apps/web/src/legacy/legacyMarkup.ts`: Legacy HTML markup extracted from the old single-file app. Avoid manual long-term edits here; move UI into React components instead.

### Legacy Public Assets: `apps/web/public`

- `apps/web/public/assets/js/questions.js`: Current static CLF-C02 question bank. Update this when changing bundled questions.
- `apps/web/public/assets/js/app.js`: Existing quiz engine, localStorage state, progress tracking, PDF viewer hooks, and legacy UI behavior. It now restores signed-in users' cloud study snapshot on startup and calls `window.StudyCouchSync` after answers, saved-question changes, important marks, notes, and check-ins.

### Backend Schema: `supabase`

- `supabase/migrations/001_initial_schema.sql`: Initial Supabase tables for profiles, study sessions, attempts, saved questions, important questions, notes, check-ins, and events.
- `supabase/migrations/002_projects_documents.sql`: Future-facing tables for projects, uploaded documents, question sets, and generated questions.
- `supabase/migrations/003_auth_profile_trigger.sql`: Creates a trigger so new Supabase Auth users automatically get a row in `public.profiles`.

### Documentation: `docs`

- `docs/ARCHITECTURE.md`: Architecture notes, platform decisions, frontend migration direction, backend responsibilities, and future AWS fit.

### Local-Only Materials

- `PDF_DOCS/`: Local source PDFs. This folder is ignored by Git and should not be pushed to GitHub.

## Documentation Rule

Every meaningful code update should also check whether documentation needs to change.

Update `README.md` when:

- A file or folder is added, removed, renamed, or moved.
- Setup, build, test, or deployment commands change.
- A feature moves from legacy JavaScript into React.
- A new environment variable is required.
- Backend tables, migrations, or data ownership rules change.
- The recommended platform or architecture changes.

Update `apps/web/README.md` when:

- Frontend structure changes.
- React components, feature folders, or routing are added.
- Frontend commands or dependencies change.
- Legacy code is migrated into typed React modules.

Update `docs/ARCHITECTURE.md` when:

- The product architecture changes.
- Supabase, Cloudflare, AWS, or deployment responsibilities change.
- New major features such as projects, PDF uploads, document parsing, or AI generation are introduced.

## Run Locally

```bash
cd apps/web
npm install
npm run dev
```

For local magic-link login, Supabase Auth should allow this local URL:

```text
http://localhost:5173
```

Useful checks:

```bash
npm run typecheck
npm run build
```

## Deployment Plan

1. Push this repository to GitHub.
2. Connect the repo to Cloudflare Pages.
3. Set the Cloudflare Pages project root to `apps/web`.
4. Use `npm run build` as the build command.
5. Use `dist` as the output directory.
6. Add Supabase Auth and database sync after the frontend shell is stable.
